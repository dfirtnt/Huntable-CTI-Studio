## Execution Methods

The workflow can be executed via three methods:

1. **Celery** (Production) - Background execution, fast, used by default from article page
2. **LangGraph Server** (Debugging) - HTTP API with full debugging capabilities
3. **Direct Test** (Testing) - Single agent testing from config page

See [Workflow Data Flow](docs/WORKFLOW_DATA_FLOW.md#execution-methods-celery-vs-langgraph-server-vs-direct) for detailed breakdown.
